Data Volume

  • Fundamental Building Block for sharing data
  • Part of Host filesystem
  • Get initialized during container creation
  • shared and reused among containers
  • data volumes are persistant
  • Mounted inside Containers
  • Access Control (ro/rw)
  • Creted either with Dockerfile or using -v subcommand
When a container is removed using the docker rm subcommand, the Docker engine does not remove the directory that was automatically created during the container launch time. This behavior is innately designed to preserve the state of the container's application that was stored in the directory.
